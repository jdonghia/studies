# Scopes - 22 Mar 2026

- Two-pass language
- Throws Error before running

1. Compile pass
2. Runtime pass

## Compile pass

- Maps scopes and its identifiers
- Buckets and marbles metaphor

- **Scope Manager x Compiler:**
  - Checks the existence of labels/identifiers on the current scope
  - Creates and place identifiers on the scope

E.g:

```javascript
// scope: red bucket

function foo(apple) {
  // foo: red marble; apple: blue marble

  // scope: blue bucket
  var baz; // blue marble
}
```

### Identifiers

- **function declarations**: initialized
- **var**: initialized and set to `undefined`
- **const, let**: keep uninitialized (stay on TDZ)

---

### Shadowing identifiers

E.g:

```javascript
var bar = "foo"; // outer identifier not accesible

function foo() {
  var bar = "baz";
}
```

### Function expressions

- Are only acessible on their own scope

```javascript
var bar = function foo() {
  // scope: green bucket
}; // IFEE: green marble

foo(); // Reference Error
```

### Units of Scope

**function declarations**: Hybrid Scoping
**var**: Function Scoping
**let, const**: Block Scoping

## Runtime

- Scope Manager x Runtime communication

- Target reference: LHS
- Source reference: RHS

### Hoisting

#### Function declarations

E.g:

```javascript
foo(); // can be called before declaration
bar; // undefined

apple; // TDZ error

var bar = "bar";

const apple = function sauce() {};

function foo() {
  // initialized by compiler
}
```

- Identifier not found on the current scope: act Lexically til global

```javascript
var bar = "bar";
function foo() {
  bar; // behave lexically
}
```

- Auto Global: Strict mode creates auto variable after ReferenceError
