- How pixels are displayed on the screen?

- HTML is a declarative language to create elements that will be parsed to the DOM. The elements will be displayed/created in order of declaration

1. Browser loads HTML
2. Javascript and CSS are read from the Element/tag

- DOM is a object that lives on C++ Browser runtime.

- When the DOM is created, the Layout and Render Engine will be responsible to map how pixels will be displayed and create the bitmap image to be painted on the Screen every 16ms

- CSS will also be parsed, creating a object with the elements and its properties (called CSSOM). It also has a copy of the DOM.

- The DOM, Layout and Render Engine are part of the WebCore.

- HTML is a one-time thing.

- Javascript has access to Web APIs like the DOM and Event API.

- Through Web IDL (Interface Descriptive Language), Javascript will access global properties like document. that has a hidden link to the real DOM

- document has many properties, getters and setters

- when accessing properties like textContent or value, these properties when on the Left Hand Side, they cause the view to be updated

- It is difficult to reason about view updates if we do not centralize the side-effects into a single converter function (One way data binding approach)

- The user will only change underlying data that lives on Javascript. The UI update will be cause by the underlying data change.
