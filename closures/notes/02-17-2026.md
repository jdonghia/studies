# Closure

- Function carries P.L.S.S.R.D.
- Permanent Lexical Static Scope Referenced Data
- Backpack: hidden `[[scope]]` property that holds closure's data.

- Functions remember if they were previously called

```javascript
function outer() {
  var ran = false;

  return inner;

  function inner() {
    // check ran on [[scope]]
    // variable closed over
    if (ran) throw new Error("have already ran");
    ran = true;
  }
}

var func = outer();
func();
func(); // error
```

- outer function EC has already being finished
- ran persists through calls

## Lexical Scope without closures

- Lexical behavior can happen without closures
- Closure happens when EC gone

```javascript
function outer() {
  var ran = false;

  function inner() {
    console.log(ran);
  }

  inner(); // does not use closure to find ran
}

outer();
```

## Side-effects

- Javascript closure is per-basis/scope, not per-variable.
- Non-referenced data may stick around

## Private data

e.g: Module pattern

```javascript
// factory function

function foo() {
  var count = 0;

  var publicAPI = {
    foo,
    bar,
    baz,
  };

  return publicAPI;

  function bar() {
    count++;
  }

  function baz() {
    count--;
  }
}

var api = foo();
api.count; // cannot access, private on function's scope
```
