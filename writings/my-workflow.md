One year ago, i was feeling very frustrated with my workflow. I used to use the most common dev setup: VS Code, Bash Terminal and the
default Windows experience. I didn't know about WTMs, customizable setups and have never tried any other IDE. Then i got into a awesome
content from devaslife channel on Youtube. He was building a personal portfolio using Javascript and some frameworks. He was using a
different IDE that i didn't know. First i tought he found a way to customize his VS Code more, but it was Neovim.

**1. Introduction: The Catalyst for Change**

- **The Problem:** Feeling fatigued by the standard VS Code environment, friction in daily tasks, and the repetitive nature of
  point-and-click workflows.
  - VS Code was not customizable. Was kinda customizable with plugin "APP Customize UI". But i wasnt working 100%, VS Code was not made for
    customization.
  - Diego Fernandes from rocketseat has a really good video to customize VS Code, tried that. https://www.youtube.com/watch?v=TW3KoPkuWEA
  - VS Code abstract some functionalities by the usage of JSON and plugins.

- **The Spark:** Discovering custom workflows via a YouTube video by _devaslife_
  ([Watch here](https://www.youtube.com/watch?v=bSMZgXzC9AA)), which sparked curiosity about Neovim.

* He also has a youtube video setting up his environment on neovim from stratch, which helped me a lot.
* In a way he has a minimalist setup, which inspired me to have the minimum setup to work, like not using 2 monitors and using a single
  monitor

- **The Inspiration:** Watching _ThePrimeagen_ and realizing the power of Vim motions and a highly customizable environment.

* Everyone gets into one of ThePrimagen videos about vim motions and how good neovim is.
* I watched his entire playlist of teaching vim motions.

- **The Core Philosophy:** A customizable environment tailored to your specific work style directly improves focus and efficiency.

**2. The Operating System and Window Management Journey**

- **The Frustration with Windows:** \* Was forced to use it for work.
  - Lack of native, reliable tiling window managers.
  - Tried _GlazeWM_ and _Komorebi_ (experienced startup/reload bugs, ultimately stuck with GlazeWM via WSL + ArchLinux when forced).
  * GlazeWM was the most stable one. BUt windows does not work well with window managers
  * Also, windows is not unix based, so its very different from MACos and linux.
  * Was using bash before WSL. WSL improves the QoL but Windows was still a friction

- **The Linux Exploration Phase:**
  - **Debian/Ubuntu:** Started with _i3_ to learn tiling window managers. Productivity increased significantly within weeks.
  - **Arch Linux:** Inspired by a _typecraftz_ playlist. Moved to _Hyprland_ for a highly customizable, from-scratch setup and stayed for
    several months.
  * typecratz has also an amazing content that helped me understand more about Arch Linux.

- **The Current State: macOS (MacBook Air M4 13")**
  - Provides a strong ecosystem that simplifies daily tasks.
  - Explored _Yabai_ (requires disabling SIP) but settled on _Aerospace_ for a clean, simple tiling window management experience.
  - **Display Optimization:** Scaled the small 13-inch display to maximize screen real estate, preferring a single screen over dual monitors
    for better focus.

**3. The Editor: Breaking Free from VS Code**

- **The Transition:** Moving from a GUI-heavy editor to terminal-based Neovim required learning bash commands and manual configuration.
- **Iteration 1: Lazyvim:** Used this pre-configured distribution initially to ease the transition while slowly learning Vim motions.
- **Iteration 2: Building from Scratch:** Moved to _Kickstart.nvim_ to understand the configuration line-by-line.
- **The Result:** A minimalist dotfiles setup with a very low plugin count. Recently added dynamic colorschemes that switch between light
  mode (for daytime energy/less eye strain) and dark mode based on the system theme.

**4. Ergonomics and Hardware: Trimming the Fat**

- **The Old Setup:** A bulky desktop computer, RGB lighting, standard keyboard, and multiple monitors with wires everywhere.
- **The Split Keyboard Rabbit Hole** (Inspired by _Primeagen_ and _Erick Wendel_):
  - _Silakka54:_ Bought a cheap, wired option on AliExpress to test the waters. Tried optimizing it down to 36 keys to minimize hand
    movement, though latency on hold-keys was an issue.
  - _Tupinikeeps Corne:_ Upgraded to a standard Corne with tenting. Tenting proved essential for relaxing the hands and preventing pain.
  - _Future Thoughts:_ Considered low-profile options like _Totem_, but sticking with Corne for its extensive customization and keycap
    options. Might revisit an ultra-minimal setup like _Ferris Sweep_ later.
- **Desk & Environment:**
  - Transitioned completely to wireless peripherals to eliminate cable clutter.
  - Added a motorized standing desk to alternate between sitting and standing.
  - Occasionally use the MacBook completely bare—relying on its excellent built-in keyboard and trackpad.

**5. Software Choices: Balancing Aesthetics and Performance**

- **Browsers:** * Tested *Zen, Arc, Firefox,* and *Chrome\*.
  - Loved the clean, tab-less sidebar navigation of Zen and Arc.
  - Ultimately abandoned them due to high battery drain and bugs (especially with Zen being open-source). Returned to _Google Chrome_ purely
    for its unmatched developer tools.
- **Terminals:**
  - Tested _Ghostty, Kitty, Wezterm, iTerm,_ and the macOS default.
  - Rejected the macOS default because the un-hideable title bar wastes precious screen space.
  - Settled on _iTerm_: Highly mature, lacks the bugs of newer terminals like Ghostty, and consumes less battery since it isn't strictly
    GPU-accelerated.
- **Note-Taking & Study:**
  - Tested _Obsidian, InkDrop, Apple Notes,_ and physical paper.
  - Used _Obsidian + Excalidraw_ on a tablet for mind-mapping (following Justin Sung's study methods), but the drawing latency felt
    unnatural.
  - Settled on pure **Markdown in a private GitHub repo**. It is fast, simple, and saves paper.

**6. Conclusion: The Perfectionist's Path Forward**

- Reflecting on the constant pursuit of the most optimized setup.
- The realization that minimalism (fewer tools, fewer wires, single monitor) yields the highest productivity.

- **Future Goal:** An ultra-clean iMac setup—just the monitor, a single power cable, Magic Keyboard, and Magic Mouse.
