# Types

- Javascript has value-types, not variables-types

- Typescript: variable-types
- Prefer Eslint?

- Primitives: undefined, null, boolean, string, number, BigInteger, Symbol.
- Non-primitive: objects

## typeof

- Always return a string.

| type      | value         |
| --------- | ------------- |
| undefined | `"undefined"` |
| BigInt    | `"bigint"`    |
| string    | `"string"`    |
| boolean   | `"boolean"`   |
| number    | `"number"`    |
| Symbol    | `"symbol"`    |
| object    | `"object"`    |
| function  | `"function"`  |
| null      | `"object"`    |
| Array     | `"object"`    |

## Null

- Historically returns typeof "object". Objects were unset to null.

## Numbers

### Special Values

#### NaN (Invalid Number)

- NaN != NaN
- Operations with NaN = NaN
- Not covered by === comparisson, must use isNaN, Number.isNaN, Object.is()

#### Negative zero (-0)

- Not covered by === comparisson, must use Object.is()

## BigInteger

- Do not mix well with numbers
- Limit determined
- Does not have limits like number type

## Undefined

### Kinds of Emptiness

#### Undefined

- Value that does not currently have a value

#### Undeclared

- Does not have been assigned yet or cannot be referenced in any acessible scope

#### Unitialized

- Remains in TDZ. Any access will throw a TDZ error

# Coercion

## Abstract operations

### ToPrimitive

- Works recursively (gets a primitive or throws an Error)

- If non-primitive, calls in order based on the hint:

| Hint   | Order                 |
| ------ | --------------------- |
| string | toString(), valueOf() |
| number | valueOf(), toString() |

String:
| value | coercion |
| ------- | ------------------- |
| {} | `"[object Object]"` |
| [] | "" |
| [[],[]] | "," |

Number:
| value | coercion |
| --------- | -------- |
| {} | NaN |
| [] | 0 |
| `[-0]` | -0 |
| `[1]` | 1 |
| `[1,2,3]` | NaN |

OBS: valueOf() { return this; }

### ToString

Primitives:

| value     | coercion |
| --------- | -------- |
| 0         | "0"      |
| -0        | "0"      |
| null      | ""       |
| undefined | ""       |
| true      | "true"   |
| false     | "false"  |
| NaN       | "NaN"    |

### ToNumber

Primitives:

| value     | coercion |
| --------- | -------- |
| ""        | 0        |
| null      | 0        |
| undefined | NaN      |
| true      | 1        |
| false     | 0        |

## Boxing

- When accessing properties, primitive turns into its non-primitive counterpart.

## Cases of Coercion

### Template String

- Implicit coercion ToPrimitive(string)

### Comparisson operators

- Implicit coercion ToPrimitive(number)

- Both double and triple check types

### Minus operator

- Prefers to use toNumber

### Plus operators

- Uses toNumber but uses toString if a least one is string

#### Double Equals

- Different types: performs coercion
- null and undefined are strictly equal

### Triple Equals

- Different types: Doesnt performs coercion
- Not sure about the types
- Protect yourself from weird behaviors

#### Double Equals x Triple

- Same type and value: they work the same
- Know your types
- Always prefer to use double equals

### Check Thruthiness

- Non-primitive

- at least one of them is a string, implicit coercion ToPrimitive(string)

- If checking true or false, allow implicit boolean coercion

### Objects comparisson

- Identity comparisson, not structural comparisson

## Explicit

- String(), Number(), Boolean() (always prefer)

- Chained operators can result in weird behaviors ( 3 > 2 > 1)

- typeof always return a string, and does not produce an error

- typeof functions for functions and object for objects
- typeof cannot check arrays, returning object. use Array.isArray()

- Embrace coercion

- Duck typing

- prefer to use null than undefined or ==

```javascript
const ref = useRef();

if (ref == null) {
}
```

- double equals coerces if type don't match

- Eslint instead of typsecript
- ToPrimtive with no hint

- check if there's any

- ++ coerces to a number before incrementing

- Compare object references, use triple equals

- avoid double equals with "" " " and 0

- if necessary, use triple equals with booleans (true or false)

- double equals only with primitive values

- less than operator with strings (alphanumeric comparisson)

- both triple and double equals check type
